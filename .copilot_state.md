# Copilot State Tracker

Current focus: expand battle/exploration systems into a shared modular map-combat flow.

Detailed backlog

1. Minimap marker layout
- Make every marker label/icon repel from nearby markers instead of visually grouping.
- Apply the same layout logic to exploration and battle minimaps.
- Keep real positions accurate while only the rendered label/icon placement drifts.

2. Cross-instance player presence
- Ensure all clients consistently know which players are targetable/present from the latest shared state.
- Fix cases where one instance updates targetable players and another does not.

3. Settings expansion
- Add a paged Settings UI.
- Add a Debug page.
- First Debug item: unlimited turn times toggle.

4. Arena PvP parity with exploration battles
- Arena 1v1 should spawn onto the world map and expose the same minimap-driven movement/chase/flee gameplay as exploration battles.
- Reuse the same backend concepts wherever possible instead of duplicating battle-specific logic.
- Keep timers working for arena PvP and make bot opponents use the configured bot Gemini model while participating in the same loop.

5. Arena preview phase
- Before arena 1v1 starts, show circular avatars for all players.
- Clicking an avatar opens that character's markdown profile in a modal.
- Give players 30 seconds to review.
- Then allow a one-time 2 minute character tweak phase before the match starts.

6. Post-implementation validation
- Rebuild after each batch.
- Keep commits incremental.
- End each iteration with vscode_askQuestions until the user says everything works perfectly.

Current known constraints
- Global exploration lock-in is the intended behavior; do not scope lock-in by location.
- Keep the direct move-state update fix that makes the mover's minimap update immediately.
- Preserve the new exploration thought-card UI, battle queued-action preview, and battle minimap work while extending the system.
